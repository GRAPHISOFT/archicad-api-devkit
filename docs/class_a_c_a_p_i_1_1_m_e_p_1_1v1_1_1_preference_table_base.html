Detailed Description

The PreferenceTableBase represents a named entity, that contains inner tables associated with cross section shapes and these table contains rows that consists of a size and a description. The rows have to be unique by value and are always in ascending order.

Since
Archicad 27

For every domain the shapes are as follows: Duct - Rectangular and Circular Pipe - Circular CableCarrier - Rectangular This means, that by adding/getting a PreferenceTable, it will already contain one or two inner shape-tables, depending on the domain. These could be empty, but you cannot add new shapes to a PreferenceTableBase, nor delete them.
